# Product Requirements Document
## Scalable Course & Practice Platform (TOEFL Edition)

### 1. Product Overview & Goals
* **Primary Objective:** Build a scalable, modular course and practice platform.
* The initial launch will focus entirely on a comprehensive TOEFL preparation course.
* The platform architecture must be flexible to support the seamless addition of 3-4 different courses in the future.
* **Core Modalities:** The system requires advanced multimedia testing capabilities, including synchronized audio playback, timed voice recording, drag-and-drop interfaces, and split-screen dynamic assessments.

### 2. User Roles & Access Control
The platform operates on a tiered, code-based access system to accommodate varying student needs and purchasing behaviors.

#### 2.1 Access Levels
* **Free Access:** Users without a code can only view selected free content (e.g., sample lessons, public announcements, sample practice tasks, and free PDF materials).
* **Premium Access (Code-Based):** Access to premium content is strictly managed via unique access codes generated by the Administrator.

#### 2.2 Access Code Logic
Codes are granular and flexible. An access code can be configured to unlock:
* An entire course (e.g., Full TOEFL)
* A specific section (e.g., Only the Reading section)
* A specific task or practice set (e.g., Only Academic Passages)
* Specific PDF materials
* **Stacking Rules:** If a student enters multiple valid codes over time, the newly unlocked content is appended to their existing account access.

### 3. Global Platform Features
#### 3.1 Communication System
The platform must feature a dual communication system to ensure accessibility.
* **Internal Messaging:** A built-in, messenger-style system allowing students to message the Admin directly from their dashboard. Must include conversation history and notifications. Admin replies directly from the platform.
* **External (WhatsApp):** A static WhatsApp contact button linked to the Admin's phone number, providing a secondary avenue for immediate contact.

#### 3.2 Materials & Practice Area
* **Embedded PDFs:** A dedicated section for lesson materials. Materials must be viewable directly within the browser (no forced downloads). The structure must support adding written text pages in later iterations.
* **Independent Practice:** A module where students can practice specific tasks without initiating a full, timed exam simulation.

### 4. Course Mechanics (TOEFL Blueprint)
The TOEFL exam consists of four locked sections. 
* **Navigation Rule:** Once a section is submitted, the user cannot return to it.
* Timers are strictly enforced; modules auto-submit or lock when the timer expires.

#### 4.1 Section 1: Reading
* **Total Estimated Time:** ~18 to 27 minutes (35-48 items).
* **UI Rules:** Split layout. The reading passage is fixed on the left side and remains visible at all times. Questions are dynamic and appear sequentially (one at a time) on the right side. Requires "Next" and "Back" navigation buttons.
* **Task 1: Complete the Words (10 Questions, ~150 sec)**
  A sentence appears with missing letters visually covered by blanks. Students type directly into the blanks. System auto-corrects immediately based on Admin inputs.
* **Task 2: Read in Daily Life (2-3 Questions per text, ~100-130 sec)**
  Short text accompanied by MCQs.
* **Task 3: Academic Passage (5 Questions per text, ~5 mins)**
  Long-form academic text accompanied by MCQs.

#### 4.2 Section 2: Listening
* **Total Estimated Time:** ~18 to 27 minutes (35-45 items).
* **UI & Logic Rules:** Audio files play ONCE only. Questions are strictly multiple-choice and must only appear after the audio concludes.
* **Task 1: Short Listening:** Single audio clip, 1 question. (20 sec to answer).
* **Task 2: Conversation:** Dialogue (30-60 sec), 2 questions. (20 sec to answer per question).
* **Task 3: Announcement:** Solo audio (30-60 sec), 2 questions. (20 sec to answer per question).
* **Task 4: Academic Talk:** Lecture (2-3 mins), 4 questions. (30 sec to answer per question).

#### 4.3 Section 3: Writing
* **Total Estimated Time:** ~23 minutes (12 items).
* **Task 1: Build a Sentence (~6m 50s):** Drag-and-drop interface. Students arrange given words into the correct order. Auto-corrected by exact match.
* **Task 2: Write an Email (7 mins):** Displays a scenario with bullet points. Includes a text editor and strict countdown timer.
* **Task 3: Academic Discussion (10 mins):** UI simulates an academic forum with a professor's prompt and student replies. The user writes their response in a text editor under a strict timer.

#### 4.4 Section 4: Speaking
* **Total Estimated Time:** ~8 minutes (12 items).
* **Recording Logic:** System recording initiates automatically after the audio prompt finishes and stops immediately when the countdown timer hits zero.
* **Task 1: Listen and Repeat (~3-4 mins, ~7 items):** Listen to an audio sentence and immediately repeat it into the microphone.
* **Task 2: Take an Interview (~3-4 mins, 4 items):** Audio prompt plays. A 45-second countdown timer begins, during which the student's answer is recorded.

### 5. Scoring & Reporting System
#### 5.1 Evaluation Engines
* **Auto-Scoring:** Immediately calculates scores for Reading, Listening, and Writing Task 1 (Build a Sentence).
* **Manual / AI-Ready Scoring:** Responses for Email Writing, Academic Discussion, and all Speaking tasks are saved to the server. These are flagged for Admin manual review, but the architecture must support API integration for AI-based grading in the future.

#### 5.2 Progress & Post-Test Reporting
Upon completion, the system generates a detailed Student Report encompassing:
* **Overall Score:** Aggregate total and individual section breakdowns.
* **Detailed Feedback:** For auto-graded tasks, displays the student's answer vs. the correct answer. For manual tasks, displays the submitted text/audio with a "Pending Review" or "Graded" status.
* **Progress Tracking:** Saves historical test attempts, allowing users to track score progression over time.

### 6. Admin Panel Capabilities
The backend dashboard must provide the Administrator with complete CRUD (Create, Read, Update, Delete) control.
* **Course Management:** Ability to add entirely new courses, sections, and structured tasks.
* **Media Uploads:** Dedicated interface for uploading PDFs and task-specific audio files.
* **Assessment Setup:** Interface to input MCQ answers, exact text strings for "Complete the Words", and correct drag-and-drop orders for "Build a Sentence".
* **Access Code Generation:** Create codes with parameters for expiration dates, specific content targeting, and usage limits. Track code usage by student.
* **Grading & Communication:** Access to view student responses, listen to saved audio recordings, input manual grades, and reply to internal platform messages.
